
## 1. 安全范围与基本原则

本文件用于约束本项目的安全与稳健性实现，目标是：
- 防止明显错误和常见漏洞
- 保证开发阶段的基本安全边界
- 保证 AI 生成代码时不引入低级安全问题
- 保证日志、配置、文件上传、外部调用等环节可控

基本原则：
1. 所有外部输入默认不可信。
2. 默认最小权限。
3. 前端校验不能替代后端校验。
4. 安全规则优先于“方便调试”。
5. 安全实现优先可解释、可检查、可测试。
6. 任何与安全相关的临时放宽都必须显式标注，不得默默长期保留。

---

## 2. 输入校验规范

1. 所有来自前端、URL、Path、Query、Body、Header、文件上传的输入都视为不可信。
2. 必须进行以下校验：
   - 空值校验
   - 长度限制
   - 格式校验
   - 枚举校验
   - 数值范围校验
3. Java 后端优先使用 Bean Validation 或等价方式。
4. 搜索、模糊查询、全文检索输入必须限制长度。
5. 排序字段、筛选字段必须做白名单校验。
6. 分页参数必须有默认值和上限。
7. AI 输出如果被当作输入使用，也不能默认可信。

---

## 3. 数据访问安全

1. 禁止拼接 SQL。
2. 数据访问优先使用 ORM / 参数化查询。
3. 不可信输入不得直接参与构造查询语句、命令、脚本。
4. 导入脚本、批处理脚本同样必须避免拼接式执行。
5. 高风险操作如删除、批量更新，应有显式条件，避免误操作全表。

---

## 4. 密钥与配置管理

1. 禁止在代码中硬编码数据库密码、JWT 密钥、API Key、云存储密钥。
2. 使用环境变量或配置模板文件管理敏感配置。
3. 仓库中只保留示例配置，不提交真实密钥。
4. 区分开发、测试、生产配置，不允许混用。
5. `.env`、本机配置、私密密钥、真实账号口令不得提交到仓库。
6. 日志、报错、截图、演示文档中也不得泄漏真实密钥。

---

## 5. 日志安全

1. 禁止在日志中输出 `password`、`token`、`secret`、`cookie` 等敏感信息。
2. 必要时进行脱敏处理。
3. 错误日志可包含模块、操作和异常摘要，但不得泄漏敏感内容。
4. 请求日志不记录完整认证信息和完整隐私字段。
5. 调试日志在合并前应清理或降级。

---

## 6. 认证与权限

1. 需要登录的接口必须校验 token。
2. token 过期、无效、缺失要明确处理。
3. 管理端接口必须进行角色校验。
4. 不同角色默认最小权限，不默认放大权限。
5. 不允许仅依赖前端判断是否有权限。
6. 测试接口、调试接口、管理接口不得默认对外开放。

---

## 7. 文件上传与文件访问

1. 限制上传类型、大小、数量。
2. 保存前校验文件后缀与 MIME。
3. 文件名处理时避免路径穿越问题。
4. 上传目录与静态资源目录尽量隔离。
5. 不直接信任客户端提供的文件名。
6. 对图片、视频、附件等资源至少做基本合法性检查。
7. 下载和访问路径不得允许任意拼接本地路径。

---

## 8. 外部接口调用

1. 所有外部 API 调用必须设置超时。
2. 异常必须兜底，不能直接把底层错误暴露给前端。
3. 失败场景要有降级或提示。
4. 调用第三方服务时不得在日志中泄漏密钥和完整敏感报文。
5. 外部返回结果必须做空值和格式检查，不能直接信任。

#### 8.1 当前外部服务边界
当前项目中的外部地图服务与外部 AI 服务均不属于当前主链路硬依赖。

因此：
1. 外部服务调用失败时，必须能够降级，不得拖垮当前主流程；
2. 路线规划主流程不得因为外部地图服务不可用而失效；
3. Diary、Recommend、Route 等基础模块不得直接依赖外部 AI 返回结果才能完成核心业务；
4. 在未最终确定厂商前，允许使用 mock / stub，但不得把 mock 逻辑误当成正式生产实现长期保留。

---

## 9. 异常与错误返回

1. 不向前端暴露堆栈详情。
2. 不允许空 catch 或吞异常。
3. 统一异常处理返回标准错误结构。
4. 用户可见错误信息保持简洁，不泄露数据库、文件系统、第三方服务细节。
5. 详细错误只进入受控日志。

---

## 10. 依赖管理与供应链基本要求

1. 新增依赖必须说明用途、版本和作用域。
2. 不随意引入大型、过时或重复依赖。
3. 构建通过后再合并进主分支。
4. 能用现有依赖解决的问题，不额外引入新库。
5. 不引入来源不清、长期无人维护的依赖。
6. AI 生成代码若新增依赖，必须显式说明风险和原因。

---

## 11. 默认安全边界

1. 默认不开启危险调试选项。
2. 默认关闭不必要的对外暴露接口。
3. 默认限制分页大小、批量大小、上传大小。
4. 默认拒绝未知排序字段、未知过滤字段、未知角色操作。
5. 默认使用最小必要的数据返回，不返回无关敏感字段。

---

## 12. 测试与验收中的安全要求

1. 测试数据尽量与真实隐私数据隔离。
2. 演示环境不得使用真实密钥和真实敏感账号。
3. 认证、上传、外部接口、异常处理至少要有最小安全测试样例。
4. 修复安全相关问题后，优先补回归测试或测试记录。

---

## 13. 禁止事项

1. 禁止硬编码任何真实密钥。
2. 禁止拼接 SQL、命令、路径。
3. 禁止在日志中输出认证信息和敏感字段原文。
4. 禁止只做前端校验、不做后端校验。
5. 禁止把调试方便放在安全规则之前。
6. 禁止把第三方接口、AI 输出、上传文件当作天然可信输入。
7. 禁止把临时放开的权限长期保留不处理。
