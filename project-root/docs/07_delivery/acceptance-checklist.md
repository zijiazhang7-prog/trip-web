# 验收检查清单（Acceptance Checklist）

## 1. 文件用途
本文件用于在项目中期检查、最终验收、答辩前，对个性化旅游系统项目的完成情况进行逐项核对，确保系统、文档、测试、演示和提交材料达到当前阶段要求。

本文件关注的是：
- 哪些内容已经完成
- 哪些内容还未完成
- 哪些内容需要在验收前重点补齐
- 当前版本是否达到“可提交 / 可演示 / 可验收”状态

本文件建议在项目后期作为总检查入口使用，并与以下文档配套维护：

- `docs/00_project/progress.md`
- `docs/00_project/risk-log.md`
- `docs/06_testing/test-report.md`
- `docs/07_delivery/demo-script.md`
- `docs/07_delivery/final-report-outline.md`

---

## 2. 使用说明

### 2.1 当前定位
当前版本为**初版验收清单模板**，适用于：
- 中期自查
- 阶段性检查
- 最终验收前总核对

### 2.2 使用方式
建议使用以下标记方式：

- `[x]` 已完成
- `[ ]` 未完成
- `[-]` 当前阶段暂不纳入
- `[!]` 有风险，需重点确认

### 2.3 使用原则
1. 不要为了“看起来完成”而勾选未完成项。
2. 若某项只完成了一部分，应在备注里写清当前状态。
3. 当前阶段不做的内容，可标为 `[-]`，但要说明原因。
4. 验收前至少完整检查一遍，并同步到周报 / 进度文档中。

## 15. 验收前最后一轮重点检查建议

验收前最后建议重点检查以下内容：

1. 是否至少有一条完整主线可稳定演示  
2. 是否所有文档都已经和当前实现一致  
3. 是否高优先级 bug 已收敛  
4. 是否演示账号、数据、路径都已准备好  
5. 是否每位组员都知道自己要讲什么  
6. 是否能清楚回答：  
   - 你们做了什么  
   - 你们怎么做的  
   - 你们的算法和数据结构体现在哪里  
   - 你们测了什么  
   - 你们还有哪些不足
