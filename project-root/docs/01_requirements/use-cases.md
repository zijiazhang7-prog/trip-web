# 用例说明（Use Cases）

## 1. 文件用途
本文件用于从用户和系统交互的角度，描述本项目的主要使用场景、参与者、目标和交互流程，为需求分析、模块划分、接口设计和测试用例设计提供基础。

本文件关注的是：
- 谁在使用系统
- 用户想完成什么任务
- 系统需要如何响应
- 主要流程是什么
- 异常或替代流程有哪些

---

## 2. 系统概述
本项目为一个面向旅游活动全流程的 Web 个性化旅游系统，覆盖以下主线场景：

1. 旅行前：选择目的地、获取推荐、搜索和筛选景点 / 学校
2. 旅行中：规划路线、查询周边设施、查看美食信息
3. 旅行后：发布旅游日记、浏览他人日记、进行内容回顾

系统同时需要体现数据结构与算法能力，包括排序、查找、图与路径规划等。课程要求中也明确包含旅游推荐、旅游路线规划、场所查询和旅游日记交流等功能。

---

## 3. 参与者（Actors）

## 3.1 普通用户
系统的主要使用者，可以注册、登录、搜索目的地、查看推荐、规划路线、查询设施、美食、发布和浏览日记。

## 3.2 管理员
负责维护基础数据和系统内容，例如目的地、场所、设施、美食、日记等数据的管理与维护。

## 3.3 AI / 智能体服务
作为系统的辅助能力提供方，用于支持创新功能，例如多模态日记生成、多人规划协商与决策说明等。

## 6. 核心用例详细说明

## UC-001 用户注册

### 目标
用户创建一个新的系统账号，以便后续使用个性化功能。

### 参与者
普通用户

### 前置条件
- 用户尚未登录
- 用户拥有可用于注册的用户名和密码

### 后置条件
- 系统成功创建用户账号
- 用户可以使用该账号登录系统

### 基本流程
1. 用户进入注册页面
2. 用户输入用户名、密码、昵称等基本信息
3. 系统校验输入格式和必填项
4. 系统检查用户名是否已存在
5. 系统创建用户账号
6. 系统返回注册成功结果

### 异常流程
- 用户名已存在 → 提示更换用户名
- 输入不合法 → 返回错误提示
- 系统异常 → 返回注册失败提示

用户登录系统，获取个性化访问能力。

### 后置条件

## UC-003 查看推荐目的地

### 目标
用户在旅行前查看系统推荐的目的地结果。

### 参与者
普通用户

### 前置条件
- 系统中已存在目的地数据
- 推荐服务可正常工作

### 后置条件
- 用户获得一组推荐结果
- 用户可继续查看详情、搜索或进入规划流程

### 基本流程
1. 用户进入推荐页面
2. 用户选择或不选择基础偏好条件
3. 系统基于热度、评分、兴趣等因素生成推荐结果
4. 系统展示前若干推荐结果
5. 用户选择某个目的地继续操作

### 异常流程
- 无匹配结果 → 返回空结果提示
- 数据加载失败 → 提示稍后重试

### 说明
课堂讨论中明确提到推荐不能单一化，应考虑多种维度，并采用“检索后二次排序”的思路。

## UC-006 单目标路线规划

### 目标
用户输入当前位置和目标点，系统生成最优路线。

### 参与者
普通用户

### 前置条件
- 用户已进入某个目的地
- 当前目的地内部图数据已建立
- 起点和终点可识别

### 后置条件
- 系统返回一条有效路径
- 用户可查看路径长度、预计时间和途经点

### 基本流程
1. 用户进入路线规划页面
2. 用户选择当前位置和目标点
3. 用户选择策略（当前 MVP 优先最短距离）
4. 系统基于图结构执行最短路径计算
5. 返回路径结果、总距离、预计时间
6. 系统在页面上展示路径和节点信息

### 异常流程
- 起点或终点无效
- 图数据不完整
- 无法到达目标点

### 说明
课程要求中明确包含“从当前位置出发到达景点或场所的最优旅游线路”，核心算法为最短路径算法。

用户输入多个目标点，系统生成整体游览路线。

### 参与者
普通用户

### 前置条件
- 用户已进入某个目的地
- 当前目的地内部图数据可用
- 多个目标点可识别

### 后置条件

### 基本流程

3. 系统输出访问顺序与总路线

### 说明

## UC-008 查询周边设施

### 目标
用户查询当前位置附近可达的设施信息。

### 参与者
普通用户

### 前置条件
- 用户位于某个目的地内部
- 当前节点位置可识别
- 设施数据和图数据已建立

### 后置条件
- 系统返回附近设施列表
- 用户可按类型过滤并查看可达距离

### 基本流程
1. 用户点击“查询周边设施”
2. 用户选择设施类型（如厕所、食堂、商店）
3. 系统基于当前位置计算可达设施
4. 系统按可达距离或可达时间展示结果
5. 用户可继续查看某设施位置或作为下一目标点

### 异常流程
- 无匹配设施
- 图数据缺失
- 当前位置无法识别

### 说明
课堂讨论中强调，周边设施应按图上的可达距离 / 时间计算，而不是简单按直线距离展示。
