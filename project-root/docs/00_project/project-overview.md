# 项目总览（Project Overview）

## 1. 项目名称
个性化旅游系统的设计与实现

## 2. 项目背景
当前用户在旅游活动中，通常会经历“旅游前选择目的地、旅游中规划路线与查询场所、旅游后记录和分享经历”这几个连续场景。传统旅游工具往往只能覆盖单一环节，例如只提供景点信息查询、只提供地图导航，或者只提供游记浏览，难以形成完整闭环。

本项目拟设计并实现一个面向旅游活动全流程的 Web 个性化旅游系统，围绕“旅行前推荐—旅行中规划与查询—旅行后日记生成与交流”构建完整功能链路。系统既要满足课程设计中的业务功能要求，也要体现数据结构课程的特点，将排序、查找、最短路径、多点路径、全文检索、压缩等数据结构与算法真正落到系统实现中。:contentReference[oaicite:2]{index=2}

## 3. 项目目标
本项目的总体目标如下：

1. 实现一个可远程访问的 Web 个性化旅游系统；
2. 覆盖旅游推荐、旅游路线规划、场所查询、美食推荐、旅游日记交流等核心功能；
3. 在系统中实现并展示排序、查找、图、路径规划等核心算法；
4. 使用智能体和 AI 工具辅助需求分析、系统设计、编码、测试和文档整理；
5. 形成可用于课程验收与演示的完整代码、文档和测试材料。

## 4. 项目定位
本项目的定位为：

- **系统形态**：Web 远程系统
- **架构方式**：B/S 架构、前后端分离、单体后端、算法模块化
- **项目类型**：课程设计项目
- **团队规模**：3 人协作
- **开发方式**：文档驱动 + 模块化开发 + AI 辅助开发

## 5. 核心功能范围

### 5.1 旅行前
- 目的地推荐
- 景点 / 学校搜索与筛选
- 基于热度、评价、兴趣等维度的排序与推荐

### 5.2 旅行中
- 单目标路线规划
- 多目标路线规划
- 周边设施查询
- 美食推荐
- 地图路径展示与结果解释

### 5.3 旅行后
- 旅游日记发布与浏览
- 日记检索与排序
- AI 辅助日记生成 / 创新功能扩展

## 7. 项目创新方向（当前版本）
在完成课程要求的基础功能之外，本项目计划结合课程要求与课堂建议，探索 1~2 个创新方向。当前阶段的候选方向包括：

1. **多模态旅游日记与路线回顾**
   - 根据用户路线、图片与文字生成更完整的旅游日记
   - 支持路线与地标联动展示
   - 强化旅行后内容沉淀与回顾体验

2. **多人旅游规划与协商决策**
   - 利用多个智能体模拟多人偏好与讨论过程
   - 输出协商方案及其理由说明
   - 由用户决定是否采纳

> 说明：当前创新功能尚处于候选与整理阶段，最终以需求文档和小组确认结果为准。

## 8. 当前模块划分（概要）
系统当前按以下模块进行划分：

- 认证模块（Auth）
- 推荐模块（Recommend）
- 路线规划模块（Route）
- 场所 / 周边设施模块（Facility）
- 美食模块（Food）
- 日记模块（Diary）
- AI 模块（AI）
- 管理模块（Admin）

模块之间通过统一的数据、查询、排序和算法能力协同工作，其中公共能力将尽量抽取为统一服务层，避免重复实现。

## 9. 当前开发原则
为保证项目可推进、可协作、可交付，当前确定以下开发原则：

1. 一次只实现一个模块或一个明确子功能；
2. 优先完成最小可运行版本，再逐步增强；
3. 文档优先于实现，先明确需求与设计，再写代码；
4. 数据库、接口、模块职责变化后必须同步文档；
5. 核心算法需与课程要求保持一致，不用黑箱能力替代最终实现；
6. 优先保证推荐—规划—查询—日记这条主线闭环跑通，再补创新功能。

## 10. 当前项目状态
当前项目仍处于**文档完善与方案收敛阶段**，尚未进入正式开发阶段。现阶段主要任务包括：

- 完善项目级文档与模块说明

- 完成数据库、接口和模块设计初稿
- 确定协作规范、代码规范、安全规范和 Git 工作方式
