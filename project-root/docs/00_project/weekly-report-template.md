# 周报模板（Weekly Report Template）

## 1. 文件用途
本文件用于统一小组成员的周报格式，便于记录每周工作内容、输出成果、问题风险和下周计划，帮助项目管理、进度追踪和答辩材料整理。

## 2. 使用原则
1. 周报关注“本周实际做了什么、产出了什么、遇到了什么问题、下周准备做什么”。
2. 周报应尽量写结果，不要只写模糊动作，例如“研究了一下”“看了看资料”。
3. 如果本周工作与文档、代码、测试、协作规则相关，应尽量写清对应文件或成果名称。
4. 如果出现明显风险或阻塞，应同步记录到 `risk-log.md`。
5. 周报建议按“个人周报 + 小组汇总”两层使用。

## 6. 填写建议
1. “已完成工作”和“产出物”尽量对应起来。
2. 周报尽量写具体文件名、模块名、接口名，而不是只写抽象描述。
3. 周报里如果提到“已完成”，最好能对应到仓库中的文档、代码或记录。
4. 一周内如果任务方向有变化，应在周报中简要说明变化原因。
5. 若使用 AI 工具完成明显工作，可在周报中注明用途，例如：
   - 需求分析
   - 模块设计
   - 接口草案
   - 代码生成
   - 测试用例整理
   - 文档润色

## 7. 推荐做法
- 每个成员各写一份个人周报；
- 小组再整理一份汇总周报；
- 每周组会后统一更新 `progress.md`；
- 出现明显风险时同步更新 `risk-log.md`。
